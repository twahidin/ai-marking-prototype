## Feature 1: Zero-Config Deployment Wizard

**Goal:** Deployers on Railway need zero manual env vars. Everything is configured through the in-app wizard.

### What changed

**`app.py`** (lines 22-87):
- `FLASK_SECRET_KEY` auto-generates on first boot and persists in `DepartmentConfig` table (key: `flask_secret_key`)
- On restart, loads from DB so sessions and encrypted API keys survive
- If env var IS set, it takes priority (backward compatible)

**`db.py`** (`_get_fernet()`):
- Falls back to DB-stored secret key when `FLASK_SECRET_KEY` env var is not set
- Enables Fernet encryption of API keys without any env var

**`ai_marking.py`** (`_resolve_api_key()`):
- Resolution chain now: session keys -> env vars -> **wizard DB keys**
- Decrypts wizard-stored keys from `DepartmentConfig` using Fernet

**`templates/setup_wizard.html`** (complete rewrite):
- 6-step wizard flow:
  1. **Welcome** - overview of what's needed with time estimates
  2. **Database check** - warns if no PostgreSQL, shows Railway setup instructions. Auto-skips if `DATABASE_URL` is set
  3. **Mode selection** - normal/department/demo cards (unchanged)
  4. **API keys** - per-provider guided walkthrough with direct links to console.anthropic.com, platform.openai.com, dashscope. Shows "Already configured" badge for env-set keys
  5. **Account setup** - app title, name, access code
  6. **Completion** - shows HOD recovery code with click-to-copy `TEACHER_CODE=xxx` for Railway env vars

### How API keys are stored
- Wizard encrypts keys with Fernet (derived from secret key) and stores in `DepartmentConfig` as `api_key_anthropic`, `api_key_openai`, `api_key_qwen`
- All routes now resolve these automatically via `_resolve_api_key()`

## Feature 2: Expanded 5-Tier Role System

**Goal:** Department mode supports HOD, Subject Head, Lead, Manager, Teacher with different permissions.

### Role Matrix

| Role | Manage Accounts | Dept Insights | Sees Accounts | Can Edit |
|------|:-:|:-:|---|---|
| **HOD** | Yes | Yes | All | All |
| **Subject Head** | Yes | Yes | All except HOD | All except HOD |
| **Lead** | No | Yes | -- | -- |
| **Manager** | Yes | No | Teachers, Leads, Managers | Teachers, Leads, Managers |
| **Teacher** | No | No | -- | -- |

### What changed

**`db.py`**:
- `Teacher.role` column widened from `VARCHAR(10)` to `VARCHAR(20)`
- Auto-migration widens column on existing Postgres databases

**`app.py`** - New constants and helpers:
- `ROLE_HIERARCHY` - numeric rank per role (hod=5, subject_head=4, lead=3, manager=2, teacher=1)
- `ROLES_CAN_MANAGE` - `{hod, subject_head, manager}`
- `ROLES_CAN_VIEW_INSIGHTS` - `{hod, subject_head, lead, owner}`
- `ALL_DEPT_ROLES` - `[teacher, lead, manager, subject_head, hod]`
- `_can_manage_accounts()` - replaces old HOD-only check
- `_can_view_insights()` - replaces old HOD-only check
- `_visible_teachers(viewer)` - filters teacher list by viewer's role
- `_can_edit_target(viewer, target)` - permission check for editing
- `_require_hod()` - now allows any managing role (HOD, Subject Head, Manager)
- New route: `POST /department/teacher/<id>/update` - inline edit name + role

**Routes updated for new roles:**
- Login redirect: managing roles go to `/department`, others to `/dashboard`
- Department dashboard, manage page, insights: role-filtered access
- Delete/revoke/purge: enforced via `_can_edit_target()`
- Assignment visibility: senior roles (HOD, Subject Head, Lead) see all
- Class visibility: senior roles see all classes

**`templates/department_manage.html`**:
- Pencil edit icon on each teacher card (inline form for name + role)
- Role dropdown respects viewer's permissions (HOD sees all roles, others see subset)
- Color-coded role badges: purple=HOD, pink=Subject Head, green=Lead, orange=Manager, blue=Teacher
- API keys section shows Configured/Not set status with masked key preview

**`templates/base.html`** (nav):
- Dashboard/Manage links: shown for HOD, Subject Head, Manager
- Insights link: shown for HOD, Subject Head, Lead
- Settings gear: shown for HOD, Subject Head
- Mark link: only shown in demo mode

**`templates/hub.html`**:
- Role-specific subtitle (Head of Department, Subject Head, Lead, Manager, Teacher)
- Hub cards adapt to role permissions
- Insights card shown for insight-capable roles

**`seed_data.py`**:
- Demo data now includes Subject Head, Lead, and Manager teachers
