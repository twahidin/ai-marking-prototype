# In-App Setup Wizard Design

## Overview

Replace env var configuration with a browser-based setup wizard that shows on first visit. Non-technical users deploy to Railway and configure everything from the browser — no env vars needed except DATABASE_URL and FLASK_SECRET_KEY (auto-generated by Railway template).

## Mode Selection (First Visit)

When DB has no `setup_complete` config record, show landing page with 3 cards:

| Card | Label | Sets |
|---|---|---|
| "I'm a Teacher" | Personal use — one teacher, your own classes | `mode=normal` |
| "I'm a Department Head" | Department — manage teachers, classes, insights | `mode=department` |
| "Demo / Showcase" | Try it out or showcase to others | `mode=demo` |

Selecting "Demo" shows follow-up: "Include department showcase with sample data?" → Yes = demo+dept, No = demo only.

## Setup Steps After Mode Selection

### Normal mode
1. App title (default "AI Feedback Systems")
2. Your name
3. Your access code (becomes TEACHER_CODE equivalent)
4. API key(s) — at least one required

### Department mode
1. App title
2. HOD name + access code
3. API key(s)

### Demo mode
1. App title (optional)
2. API key(s) — at least one required

## Data Storage

Use existing `DepartmentConfig` model (key/value, Fernet encrypted for secrets).

| Key | Example |
|---|---|
| `app_mode` | `normal` / `department` / `demo` / `demo_department` |
| `app_title` | `Math Department` |
| `setup_complete` | `true` |
| `api_key_anthropic` | (encrypted) |
| `api_key_openai` | (encrypted) |
| `api_key_qwen` | (encrypted) |

## Resolution Order

For any setting: **DB value → env var → default**

APP_TITLE, DEPT_MODE, DEMO_MODE become dynamic lookups (DB first, env fallback). API key resolution: wizard keys → assignment keys → department keys → env vars.

## Settings Page

Accessible from hub (gear icon, visible to owner/HOD only):
- App title (editable)
- Current mode (changeable with warning)
- API keys (masked, editable)
- Access code (editable, normal mode)

Route: `/settings`

## Wizard Route

`/setup/wizard` — only accessible when `setup_complete != true`. After completion, redirects to hub. All subsequent visits go to hub.

## Env Var Compatibility

Env vars still work as fallback. Technical users can skip the wizard by setting env vars — if `DEPT_MODE`, `DEMO_MODE`, or API keys are set via env, the app uses them. The wizard overrides env vars when both exist.
